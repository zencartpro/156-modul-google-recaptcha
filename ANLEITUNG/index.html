Sinn und Zweck dieses Moduls

Normalerweise reicht der in Zen Cart 1.5.6 deutsch integrierte Spamschutz zur Absicherung des Kontaktformulars aus.
Sollten Sie über das Kontaktformular des Shops trotzdem Spam bekommen, dann können Sie es mit diesem Modul einfach mit einem zusätzlichen Google reCaptcha Version 2 ("Ich bin kein Roboter") absichern.
Optional kann das reCaptcha auch auf den Seiten Registrierung und Artikelbewertung schreiben integriert werden, falls auch über diese Seiten Spamregistrierungen oder Spambewertungen kommen.

Version 3.6.1 verwendet gegenüber früheren Versionen dieses Moduls eine neue Dateistruktur und die neueste reCaptcha PHP library
Da das reCaptcha nun nicht mehr via google.com aufgerufen wird, blockt diese Version auch Spammer, die von Rechnern z.B. aus China oder anderen Ländern operieren, in denen der Internetzugang zu google.com geblockt ist.
Diese Version enthält auch Änderungen, die die Felder Name und Email befüllt lässt ("Sticky Fields") falls das Captcha nicht angekreuzt oder nicht bestanden wird, damit der Besucher nicht alles wieder neu eingeben muss.
Außerdem neu in dieser Version: Captcha ist in mehrsprachigen Shops automatisch multilingual für alle von Google unterstützen Sprachen

Voraussetzung:
kostenlose Registrierung auf
https://www.google.com/recaptcha/admin

Wichtiger Hinweis für Benutzer des bisherigen Moduls:
Falls Sie bereits das reCaptcha Modul 3.4.0 und älter in Ihrem Zen Cart Shop verwenden, dann entfernen Sie es zunächst komplett, bevor Sie diese Version 3.6.1 installieren. Die Vorgangsweise zum Entfernen des alten Moduls ist unter "Update" beschrieben.

Installation

Voraussetzungen


Dieses Modul ist nur für Zen Cart 1.5.6 deutsch geeignet und ändert einige Corefiles.
Die mitgelieferten geänderten Dateien sind auf dem Stand von 1.5.6f, bevor Sie Ihren 1.5.6 Shop also noch weiter modifizieren, stellen Sie erst sicher, dass Sie die neueste 1.5.6f Version verwenden und führen Sie falls nötig erst ein Update auf 1.5.6f durch.

Für den Einbau dieses Moduls sollten Sie über folgende Werkzeuge verfügen:

1) Ein guter Texteditor.
Damit ist nicht das in Windows enthaltene Notepad oder Microsoft Word gemeint.
Sie benötigen einen Texteditor, der utf-8 versteht und auch im Format utf-8 ohne BOM abspeichern kann.
Meine Empfehlung: UltraEdit (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist der kostenlose Texteditor Notepad++

2) Ein Tool zum Vergleichen von Dateien
Bei der Installation dieses Moduls ist es erforderlich, dass Sie den Inhalt einiger Ihrer bestehenden Zen Cart Dateien mit dem Inhalt der neuen Moduldateien vergleichen und die Änderungen zusammenführen ("mergen").
Meine Empfehlung: BeyondCompare (kostenlose 30 Tage Testversion verfügbar)
Ebenfalls gut geeignet ist das kostenlose Programm WinMerge


Empfehlung:

Dieses Modul erst in einem Testsystem einbauen und dort konfigurieren/testen und endgültig an die eigenen Wünsche anpassen. Erst dann in einem Liveshop einsetzen! Sichern Sie unbedingt alle Dateien Ihres Shops per FTP und sichern Sie die Datenbank mit phpMyAdmin oder anderen geeigneten Tools!

WICHTIG
Vor dem Einbau dieser Änderungen:
BACKUP von Shop und Datenbank machen!
Keine Haftung, Verwendung auf eigene Gefahr!
BACKUP gemacht? Ok, dann weiterlesen...

Die Installation erfolgt in folgenden Schritten. Halten Sie diesen Ablauf ein!

1)
Falls Sie noch keine reCaptcha Registrierung haben, registrieren Sie die Domain des Shops zunächst bei Google reCaptcha:
https://www.google.com/recaptcha/admin
Wählen Sie dort als reCaptcha Typ die reCaptcha Version 2 aus und kreuzen Sie an:
Kästchen "Ich bin kein Roboter."
Anfragen über das Kästchen "Ich bin kein Roboter." validieren
Sie bekommen dann Websiteschlüssel und Geheimen Schlüssel angezeigt

Mit dem Schieberegler unter "Sicherheitspräferenz" können Sie einstellen wie streng das Captcha sein soll. Eine starke Ausrichtung zu "am sichersten" wird dazu führen, dass bloßes Anclicken der Checkbox meist nicht reichen wird und eine zusätzliche Bildauswahl im Stil von "Alle Fotos mit Hydranten" erscheinen wird.

2)
Öffnen Sie die Datei includes/functions/extra_functions/functions_google_recaptcha.php im Ordner NEUE DATEIEN mit einem Texteditor und tragen Sie Domain, Websiteschlüssel und Geheimen Schlüssel in die entsprechende Definitionen in Zeile 18 ein:

domain: statt www.yoursite.com die Domain, auf der Ihr Shop läuft, z.B. www.meinshop.de
sitekey: statt ENTER_YOUR_SITE_KEY_HERE den Websiteschlüssel
privatekey: statt ENTER_YOUR_PRIVATE_KEY_HERE den geheimen Schlüssel

Zeile 18 sieht nach Ihrer Änderung also dann ca. so aus:

$reCaptchaKeys[] = ['domain' => 'www.meinshop.de', 'sitekey' => 'abc123abc123abc123', 'privatekey' => 'abc123abc123abc123'];

3)
Nach dieser Änderung nun alle Dateien/Ordner aus dem Ordner NEUE DATEIEN in der vorgegebenen Struktur in die Zen Cart Installation hochladen.
Dabei werden keine bestehende Dateien überschrieben.

4)
Im Ordner GEAENDERTE DATEIEN den Ordner namens DEINTEMPLATE auf das im Shop verwendete Template umbennen, z.B. responsive_classic.

Wenn Sie Zen Cart 1.5.6 gerade frisch installiert haben und noch keinerlei Änderungen an den Dateien vorgenommen haben, können Sie nun alle Dateien/Ordner aus dem Ordner GEAENDERTE DATEIEN in der vorgegebenen Struktur in die Zen Cart Installation hochladen. Dabei werden dann einige Dateien überschrieben.
Wenn Sie Zen Cart 1.5.6 schon länger im Einsatz haben und schon einmal Änderungen an Dateien vorgenommen oder andere Module eingebaut haben, dann laden Sie die Dateien keinesfalls einfach hoch.
Vergleichen Sie alle Dateien im Ordner GEAENDERTE DATEIEN mit den entsprechenden Dateien in Ihrem Shop und nehmen Sie die Änderungen manuell per WinMerge oder BeyondCompare vor.
Dann spielen Sie die geänderten Dateien in der gezeigten Struktur ein.

5)
Prüfen Sie nun in Ihrem Shop, ob auf der Kontaktseite das reCaptcha erscheint und testen Sie die Funktionalität

6) OPTIONAL
Sollten Sie das reCaptcha auch auf der Registrierungsseite und/oder beim Schreiben von Bewertungen verwenden wollen, sehen Sie sich in includes/functions/extra_functions/functions_google_recaptcha.php die Zeilen 22 bis 24 an:

const GOOGLE_RECAPCHTA_CONTACT_US = 'true';
const GOOGLE_RECAPCHTA_CREATE_ACCOUNT = 'false';
const GOOGLE_RECAPCHTA_REVIEWS = 'false';

Standardmäßig ist nur die Kontaktseite aktiviert, ändern Sie auch in Zeile 23 und 24 von false auf true, falls Sie auch dort reCaptcha nutzen wollen

Im Ordner OPTIONAL finden Sie im Ordner GEAENDERTE DATEIEN die Dateien, die zusätzlich für Registrierungsseite und Bewertungsseite geändert werden müssen.
Im Ordner GEAENDERTE DATEIEN des Ordners OPTIONAL den Ordner namens DEINTEMPLATE auf das im Shop verwendete Template umbennen, z.B. responsive_classic.

Wenn Sie Zen Cart 1.5.6 gerade frisch installiert haben und noch keinerlei Änderungen an den Dateien vorgenommen haben, können Sie nun alle Dateien/Ordner aus dem Ordner GEAENDERTE DATEIEN in der vorgegebenen Struktur in die Zen Cart Installation hochladen. Dabei werden dann einige Dateien überschrieben.
Wenn Sie Zen Cart 1.5.6 schon länger im Einsatz haben und schon einmal Änderungen an Dateien vorgenommen oder andere Module eingebaut haben, dann laden Sie die Dateien keinesfalls einfach hoch.
Vergleichen Sie nun alle Dateien im Ordner GEAENDERTE DATEIEN des Ordners OPTIONAL mit den entsprechenden Dateien in Ihrem Shop und nehmen Sie die Änderungen manuell per WinMerge oder BeyondCompare vor.
Dann spielen Sie auch diese geänderten Dateien in der gezeigten Struktur ein.

UPDATE VON ÄLTEREN RECAPTCHA VERSIONEN

Falls Sie bereits das reCaptcha Modul 3.4.0 und älter in Ihrem Zen Cart Shop verwenden und auf 3.6.1 aktualisieren wollen, dann entfernen Sie das alte Modul zunächst komplett, bevor Sie diese Version 3.6.1 installieren:

Löschen Sie folgende Dateien aus Ihrer Zen Cart Installation:

includes/auto_loaders/config.google_recaptcha.php
includes/classes/observers/class.google_recaptcha.php
includes/classes/observers/google (gesamter Ordner)
includes/functions/extra_functions/recaptchalib.php
includes/languages/german/extra_definitions/reCaptcha.php
includes/languages/english/extra_definitions/reCaptcha.php

Machen Sie die Änderung rückgängig, die das alte Modul vorgenommen hat in:
includes/templates/DEINTEMPLATE/templates/tpl_contact_us_default.php
Entfernen Sie folgenden Eintrag aus dieser Datei:
<?php echo recaptcha_get_html(false); ?>

Sollten Sie das reCaptcha auch für Registrierung und Bewertung schreiben verwendet haben, prüfen Sie nun noch folgende Dateien:
includes/templates/DEINTEMPLATE/templates/tpl_modules_create_account.php
includes/templates/DEINTEMPLATE/templates/tpl_product_reviews_write_default.php

Entfernen Sie folgenden Eintrag aus diesen Dateien:
<?php echo recaptcha_get_html(false); ?>

Prüfen Sie nun ob Kontaktseite, Bewertungsseite und Registrierungsseite nun wieder problemlos ohne reCaptcha funktionieren

Und dann gehen Sie so vor wie unter Neuinstallation beschrieben.

DEINSTALLATION

Alle von diesem Modul neu angelegten Dateien vom Server löschen.
Alle von diesem Modul geänderten Dateien wieder mit den Originalversionen bzw. den Dateien Ihres Backups ersetzen.
